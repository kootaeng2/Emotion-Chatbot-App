`src/main.py` 파일의 추천 생성 로직을 다음과 같이 완전히 분리했습니다.

1.  **추천 생성 함수 분리:**
    *   `generate_recommendation_for_main`: **`main` 페이지 전용** 함수입니다. 기존처럼 추천 이유가 포함된 결과를 생성합니다.
    *   `generate_recommendation_for_diary`: **`diary` 페이지 전용** 함수입니다. 요청하신 대로 추천 이유는 빼고, 이모지만 간략하게 표시되는 결과를 생성합니다.

2.  **API 호출 수정:**
    *   `main` 페이지에서 사용하는 API (`/api/predict`)는 `generate_recommendation_for_main` 함수를 호출합니다.
    *   `diary` 페이지에 보여질 일기를 저장하는 API (`/diary/save`)는 `generate_recommendation_for_diary` 함수를 호출하여, 이모지만 포함된 추천 내용을 저장합니다.

**결과:**

이제 두 페이지의 추천 로직이 서로에게 전혀 영향을 주지 않습니다.
